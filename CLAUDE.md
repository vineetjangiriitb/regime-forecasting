## What This Project Is

A research project testing the hypothesis: **does a two-stage pipeline (HMM regime detection → regime-conditioned TFT) outperform a single global TFT on S&P 500 daily return forecasting?**

There are 6 phases, each with its own notebook and corresponding `src/` module. Phases are executed sequentially — each phase's outputs are inputs to the next.

## Running Notebooks

Execute a phase notebook non-interactively (the standard way to run a phase):
```bash
jupyter nbconvert --to notebook --execute --inplace --ExecutePreprocessor.timeout=300 notebooks/01_data_foundation.ipynb
```

Start Jupyter for interactive exploration:
```bash
jupyter notebook
```

## Critical Rules (Anti-Lookahead-Bias)

- **Macro data (FRED) must be lagged 1 day** before use as a model feature — Fed announcements are not known intraday.
- **HMM must be fit only on train data (2000–2017).** Apply the fitted model to val/test — never refit on full dataset.
- **Test set (2021–2023) is sacred** — do not load or inspect it until Phase 6.
- **Regime labels are train-set only** when fitting the HMM. Val/test labels come from applying the frozen model.

## HMM Notes (Phase 2)

- Feature inputs: `realized_vol_20d`, `yield_spread`, `vix` — standardized to zero mean, unit variance before fitting.
- Model selection: fit `GaussianHMM` for K = 2, 3, 4, 5; pick lowest BIC.
- The transition matrix diagonal **must be > 0.90** — regimes should be persistent, not flickering day-to-day.
- Regime color palette (use consistently across all plots): `{0: '#4CAF50', 1: '#F44336', 2: '#FF9800', 3: '#2196F3'}`.

## TFT Notes (Phases 4–5)

- Library: `pytorch-forecasting` with `pytorch-lightning` training loop.
- Input window: 60 days of history; forecast horizon: 1 day.
- Global baseline takes no regime input. Regime-conditioned variants: TFT-Separate (one model per regime), TFT-Conditioned (regime as static categorical), TFT-Ensemble (soft-weighted blend using HMM posteriors).
- Expected val MAE: 0.8–1.2%. Directional accuracy alert threshold: > 60% likely indicates lookahead bias.

## Evaluation (Phases 4–6)

Report all three metrics on every model: **MAE**, **RMSE**, **directional accuracy**. Always break down by regime — aggregate numbers alone are insufficient to support the hypothesis. Final significance test: Diebold-Mariano (p < 0.05 threshold), implemented in `evaluation.py`.
